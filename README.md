# 일대다 체스 (One vs Many Chess)

1명의 호스트가 여러 명의 도전자 팀과 **하나의 공유 체스보드**에서 실시간으로 체스 대결을 펼치는 멀티플레이어 게임입니다. 도전자들은 투표로 한 수를 결정합니다.

## 게임 방식

- **호스트**: 방을 생성하고 게임 시작 전 말 색상(백/흑/랜덤)을 선택
- **도전자**: 방에 참가하여 팀으로 투표를 통해 수를 결정
- **보드**: 8x8 체스보드 (단일 공유 보드)
- **승리 조건**: 상대 킹을 체크메이트

### 체스 규칙

- 백이 먼저 착수 (호스트가 백이면 호스트 선공, 흑이면 도전자 팀 선공)
- 캐슬링, 앙파상, 폰 프로모션 등 모든 정규 체스 규칙 지원
- **무승부**: 스테일메이트, 50수 규칙, 3회 반복, 기물 부족

### 프로모션 투표

폰이 프로모션 칸에 도달할 때 각 프로모션 선택(예: e7→e8=퀸 vs e7→e8=나이트)은 별개의 투표로 집계됩니다.

### 턴 진행

1. 백이 먼저 착수 (호스트가 백이면 호스트 선공, 흑이면 도전자 팀 선공)
2. **호스트 턴**: 말을 클릭한 뒤 이동할 칸을 클릭하여 착수
3. **도전자 투표**: 30초 타이머 시작 — 각 도전자가 출발 칸과 도착 칸을 클릭해 투표 (변경 가능); 프로모션 수는 프로모션 다이얼로그에서 기물 선택 후 제출
4. 실시간으로 투표 현황이 모든 플레이어에게 표시
5. 타이머 종료(또는 전원 투표 완료) 시 다수결로 수 결정; 동수이면 랜덤 선택; 무투표이면 합법 수 중 랜덤 착수

## 기술 스택

### Client

- React 18 + TypeScript
- Vite
- Raw WebSocket (Socket.io 미사용)
- 한국어 / 영어 지원 (브라우저 언어 자동 감지)
- 스탠드얼론 웹앱

### Server

- Go
- gorilla/websocket
- notnil/chess (체스 엔진)

## 실행 방법

### 서버 실행

```bash
cd server
go mod tidy
go run main.go
```

서버가 `http://localhost:3001`에서 실행됩니다.

### 클라이언트 실행

```bash
cd client
npm install
npm run dev
```

클라이언트가 `http://localhost:5173`에서 실행됩니다.

## 테스트

### 서버 테스트

```bash
cd server
go test ./...
```

레이스 디텍터 포함 테스트:

```bash
cd server
go test -race ./...
```

## 빌드

### 클라이언트 빌드

```bash
cd client
npm run build
```

### 서버 빌드

```bash
cd server
go build -o chess-server .
./chess-server
```
